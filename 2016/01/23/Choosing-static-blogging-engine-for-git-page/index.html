Choosing static blogging engine for git page

TL;DR

Hexo is a node.js blogging engine which promises to support git page static blogging well.

Hexo

I chose hexo as my blogging engine, because I wanted to use git pages for my blog, since the topic of posts will be about programming, mostly in Javascript.
When I check Jekyll two years ago, I couldn’t understand the git page blogging mechanism, didn’t like Ruby. But now, because I fell in love with Javascript again, on account of npm, node-debugger and my proficiency, I looked for node.js blogging engines on github. Amongst projects that have more than 100 stars(I noticed that there are much less projects than I expected), hexo was clearly saying that it supports git page blogigng well. So I tried as follows, and am satisfied so far.

Installation

When anything went wrong, hexo-cli let me know what the problem was, and told me to go to trobleshooting page. It was helpful.

npm install hexo

I already have running node.js (v4.2.1) on my OS X El Capitan. You can install hexo using npm.

github setting for git page repository

I was recommended to have a github project name username.github.com (see Jamie Paton’s blog). I don’t know why this is a good naming rule, but can’t find a reason to oppose to either.
So, you can create a new project named as above. Go to github page to crate a new project. Repository name should be *username*.github.com.

local setting for git page repository

Instead of cloning from github, you can use hexo init. I created the blog directory under ~/Documents, well, I felt it that way.

Then, I changed values of title, subtitle, url, author, deploy in my _config.yml file. Most important value was url and deploy. Again, according to Jamie Paton’s blog deploy should look like this.

Note : type should be git, not github.

Hello, hexo

Just to see what it would look like, I ran hexo server without writing any posts. You need to install hexo-server to check your blog locally (or remotely if you don’t want the git pages).

Opening http://0.0.0.0:4000/ with browser, I found that it worked as I expected.

deploy

I also wanted to check the deployment feature of hexo before I write anything. hexo-deployer-git module was required.

Checking my page on http://*username*.github.io/, well, in this case http://sangwook-kim.github.io/, was satisfactory.
Note : In some cases, you might need to set git credentials. see github help page

writing your own Hello World with Hexo.

write a post

Now, you can write(edit) your post.

preview your post

Open http://0.0.0.0:4000/ with your browser.

Where to go Next?

Now, you can read hexo page to write more articles. Or just keep writing until you want to change something or add something.
